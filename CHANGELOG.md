v0.1.0 — 2025-09-20 — Заложение фундамента: Архитектура и ядро системы

На этом этапе были приняты ключевые архитектурные решения, которые определили весь дальнейший путь развития проекта. Мы сосредоточились на создании надёжного, масштабируемого и безопасного фундамента.

    Разделение на микросервисы: После долгих обсуждений было решено отказаться от монолитной архитектуры в пользу двух независимых Telegram-ботов: user_bot и admin_bot. Это позволило изолировать клиентскую и административную логику, что критически важно для безопасности и удобства поддержки.

    Оркестрация через Docker Compose: Чтобы избежать проблем с развёртыванием и зависимостями, мы сразу внедрили Docker Compose. Это позволило стандартизировать окружение и упростить запуск всего стека одной командой.

    Единый источник данных для курсов валют: Для обмена актуальными курсами между ботами был создан общий Docker volume. Файл shared/currency_rates.json стал «единственным источником правды», исключив любую возможность рассинхронизации данных.

    Безопасная конфигурация: Все чувствительные данные (токены, ID администраторов) были вынесены в переменные окружения и .env файлы, следуя лучшим практикам "Двенадцатифакторного приложения".

    Централизованное логирование: Настроили вывод логов в stdout, что является стандартом для контейнеризированных приложений и упрощает их сбор и анализ в будущем.

v0.5.0 — 2025-09-27 — Релиз клиентского функционала: User Bot

Первый большой функциональный релиз, полностью сфокусированный на пользовательском опыте. Нашей задачей было сделать максимально простой и интуитивно понятный инструмент для клиентов.

    Интуитивное главное меню: Разработано простое меню с быстрым доступом ко всем ключевым функциям: оформление заказа, ручной расчёт, ссылки на канал, отзывы и поддержку.

    Автоматизация заказа по ссылке:

        Реализован парсинг ссылок с популярных площадок (Poizon/dw4) через внутреннее API.

        Встроен умный калькулятор, который на лету конвертирует цену в рубли по актуальному курсу и добавляет комиссию (15%, но не менее 15₽).

        Для управления ожиданиями пользователей был добавлен дисклеймер о том, что не все сайты поддерживаются.

    Гибкий ручной расчёт:

        Создан пошаговый сценарий (state-machine), ведущий пользователя от выбора валюты до получения итоговой цены.

        Поддерживаются основные мировые валюты: USD, EUR, GBP, JPY, CNY.

        Встроена валидация ввода для защиты от некорректных данных.

    Продуманный UX:

        После каждого действия пользователь возвращается в главное меню, что создаёт предскаваемый и удобный флоу.

        Все сообщения оформлены в едином стиле с использованием эмодзи для лучшего визуального восприятия.

v1.0.0 — 2025-10-01 — Первый публичный релиз: Admin Bot и запуск системы

Этот релиз знаменует собой официальный запуск проекта. Был полностью реализован административный функционал, и оба бота были объединены в единую рабочую систему.

    Надёжная авторизация в Admin Bot: Доступ к боту строго ограничен списком Telegram ID из переменной окружения ALLOWED_USER_IDS. Все попытки несанкционированного доступа логируются.

    Статистика заказов: Реализована команда /count для вывода детальной статистики по всем заказам с группировкой по пользователям.

    Массовые рассылки: Добавлен мощный инструмент рассылок с поддержкой текста, изображений и функцией предпросмотра перед отправкой. Для стабильности после каждой рассылки бот автоматически перезапускается.

    Динамическое управление курсами валют: Администратор получил возможность изменять курсы валют через интуитивное меню. Изменения мгновенно становятся доступны для user_bot.

    Автоматические напоминания: Чтобы администраторы не забывали обновлять курсы, был внедрён ежедневный планировщик, отправляющий напоминание в 10:00 по Москве.

v1.1.0 — 2025-10-07 — Хотфиксы и стабилизация

После запуска в реальных условиях мы собрали первую обратную связь и выявили несколько узких мест. Этот релиз был направлен на быстрое исправление и повышение стабильности.

    Улучшено логирование ошибок: Переработана система логирования, сообщения об ошибках стали более подробными, что в разы ускорило диагностику проблем.

    Постоянный дисклеймер: Дисклеймер о поддержке сайтов теперь показывается при каждом расчёте по ссылке, а не только в первый раз.

    Исправлены критические баги: Устранены ошибки в логике выбора валют и некорректная работа кнопок "назад" в некоторых сценариях.

    Оптимизация работы с currency_rates.json: Переписан механизм чтения/записи файла с курсами, чтобы гарантировать консистентность данных даже при высокой одновременной нагрузке.
